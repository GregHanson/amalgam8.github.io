How It Works

At the heart of Amalgam8, are two multi-tenanted services:

Controller

A multi-tenant controller that monitors the registry and updates the sidecar configuration accordingly. It provides APIs to the developer for configuring rules for request routing, fault injection, etc. These rules are then translated and sent to the sidecars belonging to the application.

Registry

A high performance multi-tenant service registry that provides a centralized view of all the microservices in an application regardless of where they are all actually running.

Sidecar

Applications run as tenants of these two services. A sidecar process automatically registers the microservice with the registry and receives routing rules from the controller. The sidecar configures the nginx proxy based on the routing rules.
